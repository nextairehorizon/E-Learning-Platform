# Module 5: Data Management for Low-Cost Sensors

## Course Information

**Duration:** 2 sessions, approximately 2 hours each

**Responsible Partner:** Know Center Research GmbH, Graz, Austria

**Instructor:** Lucas Iacono — liacono@know-center.at

## Overview

This course focuses on the management of heterogeneous data generated by low-cost sensor systems. It addresses the process of transforming raw sensor data into structured, documented, and queryable datasets suitable for data science and analytical workflows.

Participants will be introduced to the main concepts, methods, and tools required for data integration, data cleaning, preparation, documentation, and storage in relational database systems. The course covers the transition from raw flat-file sources to conceptual and logical data models, and finally to implementation in a relational database environment using PostgreSQL.

### Sub-topics
* Data integration
* Data cleaning and preparation
* Data formats
* Data descriptions and documentation
* Conceptual and logical data models
* Relational databases
* Metadata

## Course Description

Low-cost sensor environments often produce heterogeneous, incomplete, and inconsistently formatted datasets. Managing such data requires a systematic approach that combines data preparation practices with sound data modeling and database design principles.

This course explores the challenges of integrating sensor data from multiple sources and formats. It introduces participants to methods for cleaning and standardizing datasets, documenting them appropriately, and preparing them for storage in relational databases. The course also provides a practical introduction to conceptual and logical data modeling, with emphasis on Entity-Relationship design, the relational model, and SQL-based data ingestion and querying.

By the end of the course, participants will have a structured understanding of how to transform raw sensor datasets into well-organized database systems suitable for analysis and downstream applications.

## Learning Outcomes

After completing this module, participants will be able to:

### Implement the basic stages of data preparation pipelines

Participants will apply fundamental techniques for data preparation and cleaning, including:

* handling missing values
* detecting and removing duplicates
* standardizing data formats
* preparing datasets for storage in relational databases

### Transform a dataset into the relational data model

Participants will develop an understanding of:

conceptual data models, including Entity-Relationship diagrams and Modified Chen Notation
logical data models, particularly the relational model
the role of SQL in mapping tabular datasets into relational database structures

### Create a database, ingest data, and execute basic SQL queries

Using PostgreSQL and SQL, participants will be able to:
create database schemas
ingest data from prepared raw files
execute basic queries on the resulting databases

## Target Audience

This module is intended for learners interested in data management practices for sensor-generated data, particularly in contexts where data originates from low-cost, heterogeneous sensing systems. It is suitable for participants with a basic interest in data science, data engineering, environmental monitoring, or applied database systems.

## Prerequisites

Participants are expected to have:

basic familiarity with tabular data
a general understanding of data analysis workflows
No advanced prior knowledge of database systems, although prior exposure to SQL may be beneficial
