# Data Visualization — Python Session

Welcome to the Python part of **Module 5: Data Visualization**.

This package contains the materials for **Mario Dudjak's session** and is designed for learners with little or no previous experience in data science. The notebooks follow the same general style as **Module 3: Scientific Data Manipulation**: short explanations, clear examples, environmental datasets, and guided practice tasks.

The focus of this session is on **static plotting in Python** for exploratory analysis and scientific communication.

## Notebook Overview

### 1. [Static Plotting with Matplotlib and Seaborn](./1%20Static%20Plotting%20with%20Matplotlib%20and%20Seaborn.ipynb)

- Topics: line plots, bar plots, plot labels, legends, readability, time-based plots, saving figures.
- Core theme: creating clear and publication-ready static figures.

### 2. [Distributions and Relationships](./2%20Distributions%20and%20Relationships.ipynb)

- Topics: histograms, boxplots, scatter plots, correlation heatmaps, grouped bar plots, matrix-style heatmaps.
- Core theme: understanding distributions and relationships before further analysis.

## Suggested Teaching Plan

This package is intended for **one session of approximately 2 hours**.

### Session 1
- Notebook 1: Static Plotting with Matplotlib and Seaborn
- Notebook 2: Distributions and Relationships

A practical pacing is:
- **Notebook 1:** 50–60 minutes
- **Notebook 2:** 55–65 minutes

Each notebook includes short explanations, runnable code, guided practice, and a short summary.

## Learning Outcomes

By the end of this session, learners should be able to:
- create publication-quality static plots to represent temporal trends and environmental variables,
- interpret distributions of environmental measurements using histograms and boxplots,
- analyze relationships between variables using scatter plots and correlation heatmaps,
- compare grouped values using bar plots and heatmaps.

## How to use these notebooks?

1. Upload the files to the appropriate GitHub folder of the E-Learning Platform repository.
2. Open the notebooks in GitHub or Google Colab.
3. Encourage learners to run each cell and modify the examples.
4. Use the guided practice tasks for classroom work or self-study.
