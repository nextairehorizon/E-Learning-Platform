# Module 1: Git version control system

This module is designed to introduce young scientists and industry employees to the crucial practice of version control using Git, shifting away from manual file backups and uncoordinated code development. It focuses on the practical workflow of tracking every change to code, data analysis scripts, and documents, ensuring reproducibility and a complete audit trail for air quality projects. Students will learn to collaborate effectively by using Git platforms to manage feature development with branching, resolve conflicts during merging, and contribute to shared projects via pull requests, establishing a foundation for modern software development and research practices.

## Learning Outcomes
By the end of this module, students will be able to:

1. Repository Setup and Management
- Initialize a new local Git repository from scratch and clone existing remote repositories to a local machine.
- Configure essential global Git settings (username and email) required for tracking contributions.
2. Core Version Control Workflow
- Stage specific file modifications using git add and commit changes with clear, descriptive messages to create a transparent project history.
- Maintain a clean project audit trail specifically for code, data analysis scripts, and document tracking in air quality projects.
3. Isolated Development
- Create, navigate, and manage separate lines of development using Git branches to work on features or data models without disrupting the main codebase.
4. Collaborative Engineering
- Effectively utilize pull and push requests for the purpose of working with remote repositories
